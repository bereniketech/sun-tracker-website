# Testing Requirements

**Rule:** Minimum 80% test coverage. All three test types required: unit, integration, E2E.

**Rule:** Write tests before implementation (Red/Green/Refactor). Follow `tdd-workflow` skill for the full protocol.

## Troubleshooting Test Failures

1. Use **tdd-guide** agent
2. Check test isolation
3. Fix implementation, not tests (unless tests are wrong)
